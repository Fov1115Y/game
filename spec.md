# 大便超人网页小游戏 Spec

## 背景
用户希望开发一个参考 4399 小游戏《大便超人》玩法风格的网页小游戏。游戏主角是一个穿红色紧身衣的卡通角色，站在画面一侧，对面的目标是一个立牌。玩家可以自定义立牌上的名字，并通过不同方式让角色向立牌发射夸张的粪便弹幕，形成恶搞、轻量、可直接游玩的小游戏体验。

当前项目目录基本为空，因此首版按“单文件网页 MVP”设计，优先复刻类似 4399 小游戏的基础可玩闭环，而不是复杂素材、关卡系统或发布平台接入。

## 目标
- 提供一个可直接在浏览器打开的单文件网页小游戏。
- 参考 4399《大便超人》的核心玩法氛围：角色面对立牌发射恶搞弹幕。
- 允许玩家输入并修改立牌上的名字。
- 展示红色紧身衣角色、对面立牌、发射动作、命中反馈和基础计数。
- 支持多种发射方式，形成简单但有变化的玩法。
- 保持整体表现为夸张卡通恶搞风格，避免真实血腥、仇恨、骚扰或成人化表达。

## 功能需求
- F1: 游戏应提供一个主画面，画面中包含红色紧身衣卡通角色、对面的立牌、背景区域和基础操作区。
- F2: 游戏应允许玩家输入或修改立牌上的名字，并在确认后立即显示在立牌上。
- F3: 游戏应提供至少三种发射方式，玩家可以选择不同方式让角色向立牌发射粪便弹幕。
- F4: 每次发射时，游戏应展示从角色到立牌方向的飞行动画。
- F5: 粪便弹幕命中立牌时，游戏应展示可见的命中反馈，例如污渍、抖动、数字增加或短文本提示。
- F6: 游戏应记录并展示命中次数或得分，让玩家能看到持续攻击的结果。
- F7: 游戏应提供重置功能，允许玩家清除立牌污渍、重置分数，并保留或重新设置立牌名字。
- F8: 游戏应在桌面浏览器中无需安装依赖即可运行。

## 非功能需求
- N1: 首版应采用单文件网页形式，玩家可通过浏览器直接打开并游玩。
- N2: 游戏视觉应采用夸张、卡通、低门槛小游戏风格，避免写实、血腥、成人化或侮辱特定受保护群体的表现。
- N3: 游戏操作应简单直观，首次进入页面后无需说明即可完成改名、选择发射方式、发射和重置。
- N4: 游戏应在常见桌面浏览器中正常运行，包括 Chrome、Edge 和 Firefox 的较新版本。
- N5: 首版不依赖外部图片、音频、字体或网络资源，避免离线打开时缺失内容。
- N6: 动画和交互应保持轻量，避免明显卡顿或长时间阻塞页面。

## 不做的事
- 不做账号系统、登录、排行榜、存档或联机功能。
- 不接入广告、支付、分享、统计或任何外部平台服务。
- 不使用外部素材资源，首版只使用网页内置绘制、样式和文本实现视觉效果。
- 不实现复杂关卡、敌人 AI、剧情、道具商店或成长系统。
- 不制作写实、血腥、成人化、仇恨或针对受保护群体的内容。
- 不追求与 4399 原游戏素材、代码或关卡的完全复刻，只参考其玩法氛围和小游戏风格。

## 验收标准
- AC1: 打开游戏页面后，能看到红色紧身衣角色、对面立牌、名字输入区域、发射方式选择、发射按钮、重置按钮和命中计数。
- AC2: 玩家输入一个名字并确认后，立牌上的文字立即更新为该名字。
- AC3: 玩家可以选择至少三种不同发射方式，并且每种方式触发后都有可见差异。
- AC4: 点击发射后，能看到粪便弹幕从角色方向飞向立牌方向的动画。
- AC5: 弹幕命中立牌后，立牌出现可见反馈，并且命中次数或得分增加。
- AC6: 多次命中后，立牌上的反馈可以累积显示，让玩家看到持续攻击的结果。
- AC7: 点击重置后，命中计数归零，立牌反馈被清除，游戏回到可继续游玩的状态。
- AC8: 在未安装依赖、未联网的情况下，直接用桌面浏览器打开网页文件即可游玩。
