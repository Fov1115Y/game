# 大便超人网页小游戏 Checklist

> 每一项通过运行代码或观察行为来验证，聚焦系统行为。

## 实现完整性
- [ ] 页面主画面完整显示，包括红色紧身衣角色、对面立牌、背景、操作区和命中计数（验证：用桌面浏览器打开 `index.html`，观察首屏元素是否完整，对应 AC1）。
- [ ] 立牌名字可以被玩家修改并立即显示（验证：输入“测试目标”并确认，观察立牌文字是否立即变为“测试目标”，对应 AC2）。
- [ ] 至少三种发射方式可选且表现不同（验证：依次选择三种方式并发射，观察弹幕数量、轨迹或节奏是否有差异，对应 AC3）。
- [ ] 每次发射都有从角色到立牌方向的飞行动画（验证：点击发射，观察弹幕是否从左侧角色方向移动到右侧立牌方向，对应 AC4）。
- [ ] 命中后出现可见反馈并增加命中计数（验证：发射命中后，观察立牌抖动、污渍或提示出现，命中计数增加，对应 AC5）。
- [ ] 多次命中后反馈可以累积显示（验证：连续完成多次发射，观察立牌污渍数量或视觉痕迹是否增加，对应 AC6）。
- [ ] 重置功能恢复可继续游玩的状态（验证：命中后点击重置，观察计数归零、污渍清除、按钮仍可继续使用，对应 AC7）。
- [ ] 游戏无需安装依赖即可运行（验证：不执行安装命令，直接用浏览器打开 `index.html` 并完成基础操作，对应 AC8）。

## 集成
- [ ] 改名输入与立牌显示正确联动（验证：输入新名字并点击确认，观察输入值经过状态更新后同步到立牌文字）。
- [ ] 发射方式选择与动画生成正确联动（验证：切换不同发射方式后点击发射，观察当前选择影响弹幕数量、轨迹或节奏）。
- [ ] 发射动画与命中反馈正确联动（验证：弹幕飞到立牌方向后，观察是否触发抖动、污渍、短提示和计数增加）。
- [ ] 重置操作与状态、视觉效果正确联动（验证：产生命中反馈后点击重置，观察计数、污渍、提示和当前可玩状态是否一致恢复）。
- [ ] 连点保护与发射按钮正确联动（验证：发射动画期间快速多次点击发射按钮，观察页面不出现明显计数错乱、残留弹幕或按钮失效）。

## 编译与测试
- [ ] HTML 文件可以被浏览器解析（验证：打开 `index.html`，页面正常渲染，控制台没有阻断运行的语法错误）。
- [ ] JavaScript 初始化流程正常完成（验证：刷新页面后，发射方式选项出现，默认立牌名字和命中计数显示正确）。
- [ ] 离线运行无外部资源缺失（验证：开发者工具 Network 面板中没有外部图片、脚本、样式、字体或音频请求）。
- [ ] 样式和动画未阻塞交互（验证：连续完成改名、切换方式、发射、重置，页面无明显卡顿或冻结）。
- [ ] 项目没有需要额外安装或构建的步骤（验证：目录中只需打开 `index.html` 即可运行，不需要执行包管理器或构建命令）。

## 端到端场景
- [ ] 场景 1：首次打开并完成基础攻击流程（验证：打开 `index.html` → 输入“路人甲”并确认 → 选择“连续多发” → 点击发射 → 观察多发弹幕飞向立牌 → 立牌出现污渍和提示 → 命中计数增加）。
- [ ] 场景 2：切换全部发射方式并观察差异（验证：分别选择“单发直线”“连续多发”“抛物线”并各发射一次，观察三种方式的数量、轨迹或节奏存在可见差异）。
- [ ] 场景 3：累积命中后重置（验证：连续发射多次 → 观察立牌污渍累积、命中计数增加 → 点击重置 → 观察计数归零、污渍清除 → 再次点击发射仍可正常游玩）。
- [ ] 场景 4：离线本地运行（验证：不安装依赖、不联网、不启动服务器，直接用桌面浏览器打开本地 `index.html`，完整完成改名、发射、命中和重置流程）。
